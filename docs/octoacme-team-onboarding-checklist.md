# OctoAcme Team Onboarding Checklist

## Purpose
Provide a standardized onboarding checklist for new cross-functional team members joining OctoAcme projects. This ensures consistent knowledge transfer and helps new members quickly understand roles, processes, and communication norms.

## Scope
Applicable to all new team members joining project teams, including developers, designers, analysts, QA engineers, and other specialized roles.

---

## Week 1: Orientation & Access

### Administrative Setup
- [ ] Complete HR/onboarding paperwork
- [ ] Set up company email and communication accounts (Slack, Teams, etc.)
- [ ] Obtain necessary hardware and software licenses
- [ ] Get added to relevant distribution lists and team channels

### Access & Accounts
- [ ] GitHub organization access granted
- [ ] Added to relevant repositories
- [ ] Project board access (GitHub Projects or similar)
- [ ] Access to documentation repositories (.copilot/ spaces if applicable)
- [ ] CI/CD and deployment tool access (as appropriate for role)
- [ ] Monitoring and observability tools access

### Initial Meetings
- [ ] Meet with Project Manager for project overview
- [ ] Meet with Product Manager to understand product vision
- [ ] Attend team introduction meeting
- [ ] Shadow a daily standup
- [ ] Review current sprint/iteration goals

---

## Week 2: Process & Documentation

### Core Documentation Review
- [ ] Read [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Review [Roles and Personas](octoacme-roles-and-personas.md) - identify your role and key collaborators
- [ ] Understand [Project Initiation Guide](octoacme-project-initiation.md)
- [ ] Study [Execution & Tracking](octoacme-execution-and-tracking.md) practices
- [ ] Review [Risks & Communication](octoacme-risks-and-communication.md) protocols

### Project-Specific Context
- [ ] Read current project charter or one-pager
- [ ] Review project roadmap and current milestone
- [ ] Understand success metrics and Definition of Done
- [ ] Review risk register and known dependencies
- [ ] Familiarize with project board workflow and column definitions

### Development Environment (Technical Roles)
- [ ] Clone relevant repositories
- [ ] Set up local development environment
- [ ] Run build and test suites successfully
- [ ] Review branching strategy and PR conventions
- [ ] Understand CI/CD pipeline stages

---

## Week 3: Active Participation

### Team Integration
- [ ] Actively participate in daily standups
- [ ] Attend sprint planning or backlog refinement
- [ ] Join one-on-one meetings with direct collaborators
- [ ] Begin contributing to team discussions and decisions

### First Contributions
- [ ] Complete a small, well-defined task appropriate for your role
- [ ] Submit first pull request or deliverable
- [ ] Participate in code/design review (give or receive feedback)
- [ ] Document any setup issues or onboarding gaps

### Role-Specific Tasks
- [ ] Review role interaction matrix to understand collaboration patterns
- [ ] Schedule meetings with key collaborators from other roles
- [ ] Identify mentor or buddy within your role discipline
- [ ] Clarify escalation paths and reporting structure

---

## Week 4: Full Integration

### Ownership & Accountability
- [ ] Take ownership of backlog items or user stories
- [ ] Understand how to escalate blockers or risks
- [ ] Contribute to retrospective or team improvement discussions
- [ ] Provide feedback on onboarding process

### Communication Cadence
- [ ] Comfortable with all recurring meetings (standups, planning, reviews, retros)
- [ ] Know when and how to communicate status updates
- [ ] Understand escalation procedures for different issue types
- [ ] Connected with stakeholders relevant to your role

### Continuous Learning
- [ ] Identify areas for skill development or knowledge gaps
- [ ] Schedule follow-up sessions with team leads or mentors
- [ ] Contribute to documentation improvements (process, technical, or product)
- [ ] Begin participating in cross-team initiatives if applicable

---

## Role-Specific Onboarding Additions

### For Developers
- [ ] Review code standards and style guides
- [ ] Understand testing strategy (unit, integration, E2E)
- [ ] Complete architecture overview session
- [ ] Review security and compliance requirements

### For UX/UI Designers
- [ ] Access design systems and component libraries
- [ ] Review brand guidelines and design principles
- [ ] Understand user research repository and insights
- [ ] Connect with Product Manager on design priorities

### For QA Automation Engineers
- [ ] Review existing test frameworks and tools
- [ ] Understand test coverage goals and gaps
- [ ] Access to test environments and data
- [ ] Review quality metrics and reporting process

### For DevOps Engineers
- [ ] Review infrastructure architecture
- [ ] Understand deployment processes and schedules
- [ ] Access to monitoring and alerting systems
- [ ] Review incident response and on-call procedures

### For Business Analysts
- [ ] Access to requirements repositories
- [ ] Review stakeholder map and communication plan
- [ ] Understand requirements gathering templates
- [ ] Schedule stakeholder introduction meetings

### For Scrum Masters
- [ ] Review team agreements and working norms
- [ ] Understand current team metrics (velocity, burndown)
- [ ] Identify current impediments and blockers
- [ ] Review retrospective history and action items

### For Technical Writers
- [ ] Access to documentation platforms and repositories
- [ ] Review documentation style guide
- [ ] Understand documentation release process
- [ ] Identify documentation gaps and priorities

---

## 30-Day Check-in

Schedule a 30-day check-in with your Project Manager or team lead to:
- Review progress and integration
- Identify any remaining gaps or concerns
- Gather feedback on onboarding experience
- Plan next steps for growth and contribution
